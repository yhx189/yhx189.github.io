Android Network measurement

This project measures the latency and bandwidth on Android networks. The latency and bandwidth of any remote server is measured by first mile and second segment connection. First mile measurement is implemented as an Android app that sends ICMP packet to the closes DNS server based on its geo-location. Latency is measured as the interval between packet trains.

First mile app can be downloaded here

Second Segment measurements focuses on the latency from the DNS server to the requested destination. It is also done by sending ICMP packets and compute the interval between the request packets and the response packets. The bottleneck here is that we do not have control over the DNS server and cannot send packet from there. We bypassed this problem by setting CDN servers from all over the world and send requests from one of the most closest CDN server based on their geo-locations. The results turned to be accurate by our testes on Alexa top 100 websites.
